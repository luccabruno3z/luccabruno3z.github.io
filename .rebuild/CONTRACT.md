# Legión de Hierro — Stats Tracker · Rebuild Contract

Single source of truth for rewriting the vanilla-JS frontend (`web/js/app.js`, `index.html`)
without breaking integration with the pre-generated JSON data files under `graphs/`.

- Source files: `web/js/app.js` (2439 lines), `index.html` (614 lines), `web/css/styles.css`, `sw.js`, `manifest.json`.
- Data files are static, generated by `graphs/PRstats_multiclans.py` and a separate demo pipeline. The frontend is read-only; it never writes data.
- Site is hosted on GitHub Pages at `https://luccabruno3z.github.io`. All data URLs are **absolute** to that origin (see §1 URL constants), so the app fetches its own data even when run from a different host.

## 1. Data contract

> **Type note:** Despite earlier appearances, all numeric fields in these JSON files are stored
> as **proper JSON numbers** (int/float), NOT strings. app.js relies on this — it calls
> `.toFixed()`, arithmetic, and `(x || 0)` directly on the values with no `Number()` coercion.
> The rewrite **must keep emitting real JSON numbers**; emitting numbers-as-strings would break
> every `.toFixed()` call and every sort/arithmetic.

#### 1.3.1 Constantes compartidas (fuente única)
`norm_caps` y `min_rounds` los publica **el scraper** (`scraper/config.py`: `NORM_CAPS`
+ `LOW_ROUNDS_THRESHOLD`) y son la **fuente única** para los tres componentes:
- **Scraper:** dueño; los escribe en `tier_config.json`.
- **Web:** `loadTierConfig()` hace `Object.assign(NORM_CAPS, norm_caps)` (binding vivo) →
  `predictor.js`/`utils.js` los usan; `rankings.js` lee `min_rounds`. `web/js/config.js`
  (`NORM_CAPS`, `MIN_ROUNDS_FOR_RANKING`) queda como **fallback** si falta el JSON.
- **Bot:** `bot/config.py` `MIN_ROUNDS` como fallback (`clan_top_benchmark`).
⚠️ Si cambia el valor, tocar **solo** `scraper/config.py`; los fallbacks deben coincidir.

### normalizeName() — CRITICAL for per-player filenames
```js
function normalizeName(name) {
  return name.replace(/[^a-zA-Z0-9_-]/g, '_');
}
```
- Replaces **every** character that is NOT `[a-zA-Z0-9_-]` with a single underscore `_` (one underscore per offending char, no collapsing).
- Case is **preserved** (no lowercasing).
- Used to build URLs for `history/` and `player_rounds/` files. The generator MUST use the identical transform when naming those files. Examples: `juan*ARG*` → `juan_ARG_`; `__1___Juanchi__` stays; `RIM:LA`-style colons → `_`.

### 2.7 Match Predictor 8v8 (`#match-predictor`)
- Predicts win probability between Team A and Team B (1..8 players each).
- Data: `playersData` + `tierConfigData.predictor_weights`.
- DOM: inputs `#teamA-p1..8`, `#teamB-p1..8`, suggestions `#suggestions-teamA-pN`/`#suggestions-teamB-pN`, form `#prediction-form`, results `#prediction-results`.
- Functions: `performPrediction`, inner `teamComposite(team)`. Composite per team = `wPS*avgPS + wKD*(avgKD/NORM_CAPS.kd) + wKPR*(avgKPR/NORM_CAPS.kpr)`, where weights come from `predictor_weights` with the `winrate` weight dropped and `{ps,kd,kpr}` re-normalized to sum 1 (fallback `{ps:.50,kd:.30,kpr:.20}`). Probabilities = each composite / sum. Animated bar fill via double `requestAnimationFrame`.

## 4. Computed logic to preserve (exact)

> Reproduce these byte-for-byte; rankings/badges/predictions depend on them.

### tierBadge(score) — thresholds from `tier_config.json`, fallback hardcoded
```
t = tierConfigData?.thresholds || {elite:0.70, veterano:0.55, experimentado:0.40, soldado:0.25}
score >= t.elite          → 🥇 Elite          (tier-elite)
score >= t.veterano       → 🥈 Veterano       (tier-veterano)
score >= t.experimentado  → 🥉 Experimentado  (tier-experimentado)
score >= t.soldado        → ⚔️ Soldado        (tier-soldado)
else                      → 🔰 Recluta        (tier-recluta)
```
`tierEmoji(score)` = `tierBadge(score).emoji`. `rankMedal(pos)`: 1→🥇, 2→🥈, 3→🥉, else `#pos`.

### classifyPlaystyle(kd, kpr, dpr, rounds) — ordered cascade, returns {emoji,name}
```
kd>=2.0 && kpr<3.0                    → 🎯 Francotirador
kpr>=5.0 && dpr>=4.0                  → 🗡️ Asesino
dpr<2.5 && kd>=1.2                    → 🛡️ Superviviente
rounds>=500 && 0.8<=kd<=1.8          → ⭐ Veterano
kpr>=4.0 && rounds>=200              → 🏋️ Tanque
rounds<50                            → 🌱 Novato
else                                 → ⚔️ Soldado
```

### getPlayerArchetype(player) — prefer stored `player.archetype` {name,emoji,desc}; else fall back to `classifyPlaystyle(K/D Ratio, Kills per Round, deaths/rounds, Rounds)`.

### experienceBadge(rounds)
```
>=1000 →🎖️ Leyenda · >=500 →⭐ Veterano · >=200 →⚔️ Experimentado · >=50 →🛡️ Regular · >=10 →🌱 Novato · else →❓ Sin datos suficientes
```

### sampleReliability(rounds) — {emoji,text,cssClass}
```
>=200 →🟢 Alta confiabilidad (reliability-high)
>=50  →🟡 Confiabilidad media (reliability-medium)
>=10  →🟠 Baja confiabilidad (reliability-low)
else  →🔴 Datos insuficientes (reliability-none)
```

### activityIndex(rounds, spr, kpr) — 0..100 int
```
volumeLog  = min( log10(max(rounds,1)) / log10(NORM_CAPS.rounds), 1 )   // NORM_CAPS.rounds=1000
engagement = min( spr / NORM_CAPS.spr, 1 )                              // spr cap 500
impact     = min( kpr / NORM_CAPS.kpr, 1 )                              // kpr cap 10
return round( (0.4*volumeLog + 0.3*engagement + 0.3*impact) * 100 )
```
Activity tier labels (display): `>=80 🔥 Muy Activo · >=60 ✅ Activo · >=40 🟡 Moderado · >=20 🟠 Bajo · else ❄️ Inactivo`.

### percentile(playerValue, allValues) → "top X%"
```
countBelow = #values strictly < playerValue
pct = countBelow/length*100 ; topPct = 100-pct
topPct<1 → "top 1%" ; else "top {round(topPct)}%"
```

### scoreBreakdown(player) → HTML, 4 progress bars
- Reads `Normalized_KD`(Combate, #00FFFF), `Normalized_Score`(Puntuacion, #FFA500), `Normalized_Kills_Per_Round`(Agresividad, #FF4444), `Normalized_Rounds`(Experiencia, #44FF44).
- Marks the min component as bottleneck (⚠️ only if its value <0.7), max as best (⭐). Values shown `.toFixed(2)`. Uses `progressBarHTML(val, 1.0, name, color)`.

### sigmoidPenalty(rounds) = `1/(1+exp(-((rounds-25)/10)))`
- Profile shows penalty warning when `penalty*100 < 95`; "rounds for 95%" = `25 + 10*ln(19)` (~`53.9`), remaining = `max(0, ceil(that) - rounds)`.

### findPlayer(data, name) — 3-priority cascade
1. exact case-sensitive `p.Player === name`
2. exact case-insensitive
3. partial `includes` (case-insensitive). First match wins; null if none.

### formatNumber(n)
- null → `'N/A'`. Non-integer number: if it equals its floor and |n|>100 → integer with `toLocaleString('es-AR')`; else 2 decimals (`minimumFractionDigits:2,maximumFractionDigits:2`, es-AR). Integers → `toLocaleString('es-AR')`. **Locale `es-AR` (dot thousands, comma decimals) is load-bearing.**

### normalizeName(name) — see §1 (regex `/[^a-zA-Z0-9_-]/g` → `_`, case preserved).

### p95(values) = sorted (nulls filtered) `[floor(len*0.95)]` or 1. Used by radar fallbacks.

### Other helpers
- `highlightWinner(v1,v2,higherBetter=true)` → `{class1,class2}` of `compare-winner`/`compare-loser`/`compare-tie`.
- `advantagePct(v1,v2)` → `"+25.3%"`; v2==0 → `+∞%`(if v1>0) else `0%`; sign-prefixed, `.toFixed(1)`.
- `progressBarHTML(value,maxValue,label,color)` → clamped 0..100% bar with title.
- `escapeHtml(value)` (escapes `& < > " '`) — applied to user/demo-derived names before innerHTML. `prettifyToken(token)` replaces `_`→space then escapes (for map/gamemode display).
- `clanLogoHTML(clan,size)` → `<img src="logos/Logo_${clan}.png" ...onerror→logos/Logo_${clan}.gif>`.

### Constants
`MIN_ROUNDS_FOR_RANKING=50` (rankings qualification), `NORM_CAPS={kd:5.0,spr:500.0,kpr:10.0,rounds:1000.0}` (activityIndex + predictor kd/kpr normalization), `CACHE_TTL=300000`, `AUTOCOMPLETE_DEBOUNCE_MS=150`.

### cachedFetch(url, {ttl=CACHE_TTL, silent=false}) — 3-layer behavior
1. In-memory `memoryCache` Map (`{data,time}`): return if `Date.now()-time < ttl`.
2. Else `fetch`; on `!ok`: `silent`→return null, else throw. On success store in memoryCache, return.
3. On network error: return **stale** memoryCache if present; else `silent`→null; else throw.
- **Note:** `cachedFetch` does NOT use localStorage. localStorage caching is a separate path inside `loadData()` only (keys `playersData`, `clanAveragesData`, `playersDataTime`; 5-min TTL; wrapped in try/catch for quota; also mirrors into memoryCache). `loadData` uses raw `fetch` for the two core files, not `cachedFetch`.

## 5. External integrations & gotchas

- **Chart.js CDN:** `<script src="https://cdn.jsdelivr.net/npm/chart.js">` in `<head>` (NOT deferred / version-pinned). Global `Chart`. Rewrite must keep Chart.js available before chart builders run.
- **Fonts:** Google Fonts `Bebas Neue` + `Roboto` (preconnect). Theme color `#00FFFF`.
- **Plotly chart links (21 clans + all_players):** static `<a>` in `#charts` → `https://luccabruno3z.github.io/graphs/<CLAN>_interactive_chart.html`. The HTML files are self-contained Plotly exports generated by `graphs/PRstats_multiclans.py`. `clan_averages_chart.html` (4.5MB) exists but is **not linked**. `RIM:LA_interactive_chart.html` has a literal colon in the URL/filename.
- **Absolute data URLs:** all fetches hit `https://luccabruno3z.github.io/...` regardless of host. Changing the domain requires updating `BASE_URL`. Logos are **relative** (`logos/Logo_<clan>.png`) with `.gif` fallback.
- **localStorage keys:** `playersData`, `clanAveragesData`, `playersDataTime` (set/read in `loadData`). 5-min TTL. Quota errors swallowed. Other data (tier config, demos, leaderboards, rounds, history, player_rounds) is only in-memory cached.
- **Service worker `sw.js`** (`CACHE_NAME="ldh-stats-v2"`): registered at `/sw.js`. Network-first, same-origin GET only, caches successful responses, falls back to cache offline. No precache. Bumping data may require bumping `CACHE_NAME` to evict stale offline copies. Registered from inline script in index.html.
- **PWA `manifest.json`:** name/short_name/icons/theme `#00FFFF`/bg `#0a0a0f`, `start_url:"/"`, display standalone.
- **Lazy-loading:** `#leaderboards` → `renderLeaderboard`, `#recent-matches` → `loadRecentMatches` via IntersectionObserver (threshold 0.1); fallback calls immediately if no IO. Fade-in observer adds `.visible` to `.section,.hero,section`. Hero counters observer (0.3). `requestIdleCallback` preloads `loadData`.
- **5-min cache (CACHE_TTL):** both memoryCache and localStorage. Stale data possible for up to 5 min after a regeneration; expected behavior.
- **Type gotcha:** all JSON numerics are real numbers; app calls `.toFixed()`/arithmetic directly. Emitting strings would break the UI. Keep types as documented in §1.
- **Case-sensitivity:** main-data keys are exact strings with spaces/slashes (`'K/D Ratio'`, `'Performance Score'`, `Player`, `Clan`). Player search `findPlayer` is case-tolerant (cascade); clan filters use `.toLowerCase()`; demo/leaderboard lookups case-insensitive on `ign`. Rankings clan dropdown values are lowercase and matched case-insensitively against `Clan`.
- **getDemoInfo loose wiring (§1.5):** current `player_details.json` records key the name as `ign`, which `getDemoInfo`'s array branch does not probe — profile demo enrichment is effectively inert today. If the rewrite wants enrichment to work, add `ign` to the candidate keys.
- **Half-built UI:** the Dashboard/bento (`#tile-*-body`), `#hero-featured`, and `#last-updated-time` have DOM + skeletons but **no JS**. Either implement or preserve placeholders; do not assume they currently render data.
- **`WD` clan:** appears in rankings dropdown and has `WD_players.json` + `WD_interactive_chart.html`, but is **absent from `clan_averages.json`** (21 clans there). Filtering rankings to WD works off `playersData`.
- **escapeHtml usage is inconsistent:** demo/leaderboard/rounds paths escape names; the main `playersData` profile/comparison/ranking/predictor paths inject `player.Player` directly into innerHTML **without** escaping. Preserve current escaping where present; consider tightening in rewrite (names like `juan*ARG*` are common, `<`/`&` rare but possible).
